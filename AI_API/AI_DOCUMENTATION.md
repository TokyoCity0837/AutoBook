# AutoBook AI — Внутрішня Документація Архітектури

> **Версія мікросервісу:** `2.1.0`  
> **Версія аналізатора:** `StyleAnalyzer v4.0 / StyleConsistencyAnalyzer v4.0`  
> **Стек:** Python · FastAPI · spaCy · TextBlob · Ollama (llama3.2)

---

## Концепція та Ключове Рішення

Ця версія побудована навколо ідеї **«Стильового Паспорту Авторського Світу»** — замість абстрактних числових метрик ми будуємо чіткий, людинозрозумілий портрет голосу автора з трьох компонентів:

1. **Тематична Палітра** — *що* автор описує (іменники, дієслова, прикметники)
2. **Голос Автора** — *як* автор будує думку (прислівники, сполучники)
3. **Авторські Штампи** — *чим* впізнається текст (фіксовані словосполучення)

Додатково в v4.0 з'явились два нових виміри: **Тональний аналіз** та **Аналіз складності тексту** — що дозволяє перевіряти не лише словниковий, а й ритмо-емоційний збіг розділу зі стилем автора.

## 1. Стильовий Паспорт — Структура Профілю

Після аналізу текстів автора система зберігає профіль такого вигляду:

```json
{
  "style_words":        [...],   // ADV, SCONJ, CCONJ — голос автора (до 40 слів)
  "thematic_words":     [...],   // NOUN, VERB, ADJ   — атмосфера та тема (до 80 слів)
  "signature_phrases":  [...],   // 2-3-грами — авторські штампи (до 30 фраз)
  "vocabulary":         [...],   // топ-50 тематичних слів — для публічного профілю
  "characters":         [...],   // витягнуті власні імена (до 20)
  "tone_metrics": {
    "polarity":    0.12,         // [-1.0 негатив ... +1.0 позитив]
    "subjectivity": 0.45         // [0.0 факт ... 1.0 суб'єктивність]
  },
  "complexity_metrics": {
    "avg_sentence_len":  18.4,   // середня кількість слів у реченні
    "lexical_density":   0.52,   // частка змістовних слів (content words)
    "sentence_variance": 7.8     // стандартне відхилення довжини речень
  }
}
```

Кожне слово у словнику має поля: `word`, `score` (TF-IDF), `freq`, `category` (`STYLE` або `THEMATIC`).

## 2. Модулі Системи

### 2.1 StyleAnalyzer v4.0 — `/api/v1/analyze`
**Файл:** `services/style_analyzer.py`

**Алгоритм:**
1. Об'єднує всі тексти автора, ділить на чанки по 500 слів.
2. Запускає `spaCy (en_core_web_sm)` з увімкненим парсером для меж речень.
3. **Фільтрація шуму:** автоматично відкидає:
   - Власні імена (`PROPN`) → кладе їх окремо в `characters`
   - Слова, які починаються з великої літери не на початку речення (імена, що не розпізнані як PROPN)
   - Широкий список `GENERIC_FICTION_VERBS`, `GENERIC_FICTION_NOUNS`, `GENERIC_FICTION_ADJ` — слова, які зустрічаються в будь-якому тексті і не є характерними для автора
   - Надто загальні прислівники (`COMMON_ADVERBS`)
4. **TF-IDF:** для кожного слова рахується `tf * idf`, де IDF нормований по кількості чанків.
5. **Фрази:** знаходить 2-грами і 3-грами через метод `_extract_phrases()`, відкидаючи ті, що починаються/закінчуються стоп-словами.
6. **Тональність:** через `TextBlob` — `polarity` та `subjectivity` всього тексту.
7. **Складність:** рахує довжину кожного речення → `avg_sentence_len`, `sentence_variance` (StdDev), `lexical_density`.

**Повертає:** повний словниковий профіль автора.

### 2.2 StyleConsistencyAnalyzer v4.0 — `/api/v1/consistency`
**Файл:** `services/style_consistency.py`

Перевіряє, наскільки новий розділ відповідає вже побудованому профілю автора.

**Формула схожості (5 компонентів):**

| Компонент | Вага | Як рахується |
|:---|:---:|:---|
| Семантична / Фразова схожість | **35%** | Покриття топ-20 авторських штампів у розділі |
| Лексичний збіг (Cosine Similarity) | **25%** | Косинусна схожість TF-IDF профілю та TF розділу |
| Тональний збіг | **20%** | Різниця polarity + subjectivity між профілем і розділом |
| Складність тексту | **10%** | Відхилення avg_sentence_len та lexical_density від норми автора |
| Покриття персонажів | **10%** | Чи присутні топ-5 персонажів автора в розділі |

**Додатково повертає:**
- `missing_signatures` — авторські штампи, відсутні в розділі
- `outlier_words` — слова, яких автор уникає, але вони часто з'являються в розділі
- `missing_characters` — персонажі, не згадані в розділі
- `tone_deviation` — числове відхилення тону
- `complexity_deviation` — числове відхилення ритму

**Мітки результату:**
- ≥ 0.70 → ✅ Підтримує Світ та Голос Автора
- ≥ 0.50 → 🟡 Хороша консистентність
- ≥ 0.30 → 🟠 Помітні відхилення в тоні або словнику
- < 0.30 → 🔴 Тон і Світ суттєво відрізняються

### 2.3 SuggestionEngine v3 — `/api/v1/suggest`
**Файл:** `services/suggestion_engine.py`

Підказує слова та фрази під час написання тексту (inline-асистент).

**Алгоритм:**
1. Об'єднує `style_words` і `thematic_words` у пул кандидатів.
2. Аналізує `cursor_context` (останні ~500 символів).
3. Для кожного слова рахує:
   - `0.50 * tfidf_score` — базова характерність для автора
   - `+ 0.35 * context_score` — чи є схожі слова поряд у курсорі (1.0 = точний збіг, 0.5 = збіг префіксу)
   - `- 0.15 * overuse_penalty` — штраф за надмірне вживання в поточному тексті (до 3 разів)
4. Фрази: повертає лише ті авторські штампи, яких ще немає в поточному тексті.

**Повертає:** `word_suggestions` (до 12) + `phrase_suggestions` (до 8).

### 2.4 TextGenerator v3 — `/api/v1/generate`
**Файл:** `services/text_generator.py`

Генерує продовження тексту через **Ollama (`llama3.2`)** у стилі автора.

**Архітектура промпту:**
- **System prompt** містить три секції зі Стильового Паспорту:
  1. *Voice Markers* — топ-20 стильових слів (прислівники, сполучники)
  2. *World Palette* — топ-20 тематичних слів (іменники, дієслова, прикметники)
  3. *Signature Phrases* — топ-10 авторських штампів
- **User prompt** — контекст (останні абзаци) + вимога написати `N` речень.

**Параметри Ollama:** `temperature=0.8`, `top_p=0.92`, `num_predict = max_sentences * 60`.

> **Важливо:** Ми не даємо LLM ніяких метрик (довжина речення, частота ком). Замість цього — реальні слова автора. Модель сама відчуває ритм через приклади.

### 2.5 VocabularyApprovalService — `/api/v1/vocabulary`
**Файл:** `services/vocabulary_approval.py`

Керує процесом затвердження слів автором (Vocabulary Builder панель в редакторі).

**Статуси слова:**
- `PENDING` — запропоновано AI, відповідь ще не дана
- `APPROVED` → слово потрапляє у публічний словник автора
- `REJECTED` → слово більше не пропонується

**Джерела:**
- `AI_SUGGESTED` — запропоновано системою з профілю
- `MANUAL` — додано автором вручну (одразу `APPROVED`)

**Методи:**
| Метод | Що робить |
|:---|:---|
| `build_approval_queue()` | Формує чергу нових кандидатів (з `signature_words` і `signature_phrases`), яких ще немає в БД Spring |
| `enrich_with_context()` | Для кожного кандидата знаходить реальний уривок з тексту автора де слово вживається |
| `apply_decision()` | Проставляє `APPROVED` або `REJECTED` |
| `get_public_vocabulary()` | Повертає лише затверджені слова |
| `add_manual_word()` | Автор додає слово вручну |

---

### 2.6 VocabularyEnricher — `/api/v1/vocabulary/{author_id}/enrich`
**Файл:** `routers/vocabulary.py`

Збагачує словник коротким визначенням та художнім прикладом використання через **Ollama**.

Повертає `VocabularyWord` з полями: `meaning`, `usage_example`, `ai_generated=True`.

---

### 2.7 ProfileStore
**Файл:** `services/profile_store.py`

Локальне файлове сховище профілів (JSON).  
Формат файлу: `data/profiles/{author_id}_v2.json`.  
Симулює реляційну таблицю для інтеграції зі Spring Boot.

## 4. Технологічний Вибір — Чому spaCy + TF-IDF + TextBlob?

| Критерій | Наш гібридний підхід | Важкі ML (Embeddings) |
|:---|:---|:---|
| **Швидкість** | ✅ < 100ms для аналізу | ❌ Повільно |
| **Зрозумілість** | ✅ Автор бачить *конкретні слова* та *числа відхилень* | ❌ Чорний ящик |
| **Нові виміри (tone, complexity)** | ✅ TextBlob + лічильники речень | ❌ Потребує окремих моделей |
| **Інтеграція з БД** | ✅ JSON → SQL без проблем | ❌ Вектори потребують pgvector |
| **Ресурси** | ✅ CPU, без GPU | ❌ GPU або платні API |
| **Генерація** | ✅ Ollama (llama3.2) локально | — |

> **Стратегія:** Класичний швидкий NLP для аналізу (< 100ms) + потужні LLM лише там, де потрібна творчість (генерація, збагачення словника).

## 5. Запуск

```bash
# Встановлення залежностей
pip install fastapi uvicorn spacy textblob httpx
python -m spacy download en_core_web_sm
python -m textblob.download_corpora

# Запуск мікросервісу
uvicorn main:app --reload --port 8000

# Swagger UI
http://localhost:8000/docs

# Перевірка здоров'я
http://localhost:8000/health
```

> Для генерації та збагачення словника потрібен запущений **Ollama**: `ollama serve` + `ollama pull llama3.2`
